### Step 1: Create a GitHub Account (if needed)
1. Go to https://github.com
2. Click **"Sign up"**
3. Enter your email, create a password, and choose a username

---

### Step 2: Create a New Repository

1. **Log in to GitHub**
2. Click the **"+"** icon in the top-right corner
3. Select **"New repository"**

4. **Fill in the details:**
   - **Repository name:** `my-memoir-app` (or any name you prefer)
   - **Description:** "My personal memoir writing app"
   - **Visibility:** Select **"Public"** (required for free GitHub Pages)
   - **Initialize:** Check âœ… **"Add a README file"**
   
5. Click **"Create repository"**

---

### Step 3: Upload Your App Files

1. **In your new repository**, click the **"Add file"** button
2. Select **"Upload files"**

3. **Drag and drop these files** (or click to browse):
   - `index.html`
   - `styles.css`
   - `app.js`
   - `README.md` (optional)
   - `INSTALLATION.md` (optional)

4. **Scroll down** and click **"Commit changes"**

---

### Step 4: Enable GitHub Pages

1. Click on **"Settings"** tab (in your repository)
2. Scroll down and click **"Pages"** in the left sidebar
3. Under **"Source"**:
   - Select **"Deploy from a branch"**
   - Choose **"main"** branch
   - Keep folder as **"/ (root)"**
4. Click **"Save"**

5. **Wait 1-2 minutes** for deployment

---

**Example:**

## ðŸ†˜ Troubleshooting

### Problem: "404 - Page Not Found"
**Solutions:**
- Wait 5 minutes after enabling Pages
- Check the URL is correct (username and repo name)
- Make sure repository is Public
- Verify `index.html` is in the root folder

### Problem: "Page is blank"
**Solutions:**
- Check all 3 files are uploaded (index.html, styles.css, app.js)
- Make sure files are in the root folder, not in a subfolder
- Clear your browser cache (Ctrl+Shift+R)

### Problem: "Can't enable Pages"
**Solutions:**
- Make sure repository is Public
- Verify you have a verified email on GitHub
- Try using a different browser

### Problem: "Changes not showing"
**Solutions:**
- Wait 2-3 minutes after committing
- Clear browser cache
- Try opening in incognito/private mode
